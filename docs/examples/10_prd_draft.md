## PRD Draft

### Opportunity Selected

家庭长途补能与停靠编排系统

### Target User

- 三口及以上家庭用户
- 假期跨城或周末 300km+ 自驾用户
- 对“少排队、少绕路、少争论、少临场决策”敏感的用户

### Core Scenario

用户出发前设定同行成员（成人/儿童/老人/宠物）、期望抵达时间、是否接受充电/加油、休息偏好，系统自动规划最优行程，并在途中根据拥堵、剩余电量/油量、服务区热度、家庭停靠需求动态调整。

### Functional Requirements

1. 家庭画像录入
2. 路线规划时联合计算：电量/油量、排队热度、洗手间/餐饮/母婴友好度、儿童休息窗口
3. 提供 A/B/C 三种旅程策略
4. 高峰期自动给出“少充一次/多停一次/下高速补能”的推荐
5. 接近服务区前主动提醒：剩余等待时间、可替代站点、到达后停车区建议
6. 与车内休息模式联动：停靠即切换小憩、喂养、观影、静音等场景

### Non-goals

- 不做完全自动驾驶
- 不做跨品牌开放式 OTA 平台
- 不独立建设线下充电网络

### Metrics

- 长途路线中用户手动改线率下降
- 用户主动搜索充电站/服务区次数下降
- 单次长途行程中的补能等待焦虑评分下降
- 复购和 NPS 提升

### Constraints

- 需要外部充电站/服务区实时数据接入
- 节假日高峰数据准确率要求高
- 需要处理隐私与推荐透明度问题

### Assumptions

- 用户愿意在出发前填写少量同行信息，换取旅途更低决策负担
