# Evidence Rules

在所有 workflow 中统一使用以下四类标签：

- `Evidence`：有明确来源支持的事实、数据、参数、公开材料或用户原话。
- `Inference`：基于多个 `Evidence` 推导出的判断。
- `Assumption`：为推进分析暂时成立、但尚未被证据充分验证的假设。
- `Evidence Gap`：当前无法获得或明显不足的信息缺口。

## 强制约束

- 不要把 `Assumption` 写成 `Evidence`。
- 不要把 `Evidence Gap` 写成结论。
- 不要把车主原话等同于普遍需求。
- 不要把媒体观点等同于真实用户痛点。

## 证据优先级

### 用户声音

1. 真实车主评论、用户访谈、论坛讨论
2. 社媒讨论
3. 媒体总结
4. 二手转述
5. 无来源观点

### 产品与参数

1. 车企官网、官方配置表、结构化车型数据库
2. 权威媒体评测
3. 二手整理
4. 无来源参数

### 市场信息

1. 官方统计、权威研究报告、公司公告
2. 原创媒体报道
3. 普通新闻转载
4. 社媒观点

## 写作要求

- 对每条关键结论给出来源或证据类型。
- 对证据冲突写出冲突点，不要强行合并。
- 对样本不足、时间过旧、来源单一等问题显式标注。
